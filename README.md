## 🌟 專案摘要 (Abstract)
本專案透過 **Google AI Gemini** 的指導，開發出一款以聊天為主的自製型 AI 助手 —— **Alya** 。我們採用 **LLaMA-3-8B** 作為核心大型語言模型 ，目標是填補目前長照資源端短缺的輔助型 AI 聊天助理需求。在製作過程中，我們透過微調與核心指令調整，使系統能精準達成專案設定的照護目標。

## 👥 使用者角色 (User Personas)
本系統主要服務對象包含：
* **需要數據紀錄者**：需要隨時有人幫忙紀錄身體數值的人士。
* **尋求情感支持者**：希望得到溫暖關懷的獨居老人、病人或身心疾病患者。

## 🧠 核心技術說明 (Technical Details)

### 1. 生成式對話與人格微調
透過 **LLaMA-Factory** 進行人格微調，讓 Alya 具備特定性格（如：俄系傲嬌），能根據使用者語境即時生成非固定式的對話回覆。

### 2. 主動社交機制
系統後端架設監控線程，當偵測到使用者長時間未互動時，系統會主動發起關懷對話，模擬真實人類社交行為以降低使用者孤獨感。

### 3. IOT 數值採集
運用 **Adafruit 金鑰** 實現 AI 與 IOT 設備的連結，達成生理數據的即時存入與讀取，實現自動化健康監測。

##📑 FHIR 資源規劃 (FHIR Resources) 
本專案以 **PHR (Personal Health Record)** 為核心架構 ，實作以下標準資源：

* **Patient**: 使用者的基本資料，包含姓名、生日、性別與聯絡方式 。
* **Observation**: 紀錄由 IoT 設備採集的心率、血壓、體溫等生理量測資料。
* **Device**: 描述醫療物聯網硬體資訊（如 Arduino 或感測器模組）。
* **Communication**: 紀錄 Alya 與使用者之間的關懷對話與衛教內容。

## 🔄 工作流程 (Workflow) 
1. **數值採集**：系統定時透過應用程式發起每日身體數值採集需求 。
2. **數據回報**：採集完畢後將數值回傳至 AI 系統 。
3. **AI 關懷**：AI 針對數值進行分析，並給予使用者相應的專業建議與關懷語句。
4. **異常警示**：若連動之智慧設備監測到數值異常，系統會立即發出警訊。
